# 实验章节：让人"信"的核心

解读的可信度全压在实验上。没有实验证据的方法解读 = 没有证据的断言。**实验必须单独成章、讲透，且图文并茂**（结果图保留原图）。目标：读者看完能回答"这方法到底有没有用、用在哪、提升多少、靠哪个改动"。

## 必含的六块（按顺序）

### 1. Setup（怎么训/怎么测）
- **模型与基座**：在什么 base 上做（范本：Qwen-32B base）。
- **超参精确列出**：优化器、lr、batch_size、rollout 的 G、关键阈值（范本：AdamW，lr=1e-6 恒定，batch=512，G=16，ε_low=0.2/ε_high=0.28，eval 温度=1.0/topP=0.7）。
- **评测集与指标**：在哪些 benchmark、用什么指标（范本：AIME 2024，avg@k 准确率）。
- 一句话说清"实验想回答什么问题"。

### 2. 主结果（方法整体有没有用）
- **保留原文主结果图/表**（①类，配 caption + 来源），别只说数字。
- 给出 baseline → 本方法的关键提升幅度，并解读"这个提升意味着什么"（范本：DAPO 把 Qwen-32B 训成强推理模型，在 AIME 上达到/超过对照）。

### 3. 消融（提升靠哪个改动）—— 解读的重头戏
- **逐技术贡献表/图**：每个改动各带来多少（范本的方法贡献分解）。这是把"四个改动"和"最终分数"连起来的关键。
- 解读每条：哪个贡献大、哪个主要管"稳定性"而非"分数"（范本：token-level loss 提分少但增稳；dynamic sampling 主要提效率，采样变多但因步数减少总时长反而缩短）。
- 用 diff 表呈现：`改动 | 去掉它会怎样 | 带来的提升 | 性质(提分/增稳/提效)`。

### 4. 训练动态（过程指标怎么读）
- 解释关键监控量**为什么重要、怎么读异常**（范本三个）：
  - **生成长度**：与稳定性/性能相关，长度涨给更大探索空间；但非单调，会停滞甚至下降，常与验证准确率合看判断是否恶化。
  - **奖励动态**：多数实验稳步上升；但训练集最终奖励常与验证准确率几乎不相关 → 过拟合信号。
  - **熵 / 生成概率**：熵要维持适当区间，过低=丧失探索、过高=胡言乱语/重复；Clip-Higher 解熵坍缩后，熵缓升有利。
- **保留原文的动态曲线图**。

### 5. 关键现象图（机制层面的证据）
- 把支撑某个改动的"现象图"放到对应改动旁或集中在实验章（范本：熵坍缩曲线支撑 Clip-Higher、dynamic sampling 收敛更快图、overlong filtering 稳定性图）。
- 每张图配一句"它证明了什么"。

### 6. 一句话结论
- 实验整体告诉我们什么（范本：四个改动各自带来提升，组合起来解决了 RLHF 训练的稳定性与有效性）。

## 写实验的硬规矩
- **每张结果图都配解读**："图里看到 X → 说明 Y"。光放图不解读 = 没讲。
- **数字全部溯源原文**，不准约等于或脑补。拿不到的数据不重画。
- **区分"提分 / 增稳 / 提效"**：很多改动不是为了涨点，而是为了稳定或省算力——不点明会误导读者。
- **消融是灵魂**：读者最想知道"提升到底靠哪个"，这块不能省、不能含糊。
