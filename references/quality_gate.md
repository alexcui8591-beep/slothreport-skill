# 质检门：交稿前逐条自检

每条都要能答"是"。任何一条"否"→回去补，不要交。

## A. 解释质量（最重要）
- [ ] **前置讲透了吗?** baseline 的机制、损失函数、一个跑通的例子都齐了，读者能拿它去 diff 主角。
- [ ] **每个改动都有 diff 表**（`维度 | baseline | new | 为什么`），且"为什么"是真原因不是复述。
- [ ] **每个公式都拆了**：整体写出 + 关键符号逐个解释 + 求和/归一方向性捋一遍。没有"甩个公式不解释"。
- [ ] **每个新概念先给直觉**（什么是X→两极→后果→解法），再上数学。
- [ ] **至少一个端到端的具体数字例子**（像 2+3*4 那样跑通），主体里每个难点尽量配数字推演或类比。
- [ ] 一个没读过原文的人，看完能复述"改了什么/为什么/好在哪"。

## B. 忠实性（不准脑补）
- [ ] **所有公式来自原文**（最好走 LaTeX 源），不是手抄走样。
- [ ] **所有数字精确**：超参（lr、batch、G、ε_low/high、温度、topP）、阈值、实验结论都对得上原文。
- [ ] 自己推断/补充的地方**显式标注**"（笔者理解）"，没伪装成原文。
- [ ] 引用的原文图标了来源；亲测实验如实说明设置，没装跑过。

## C. 图文并茂 & 不丢原图
- [ ] **论文的关键原图都在**：架构图、结果曲线、消融图、机制示意、定性样例——一张没漏，配 caption + 来源。
- [ ] 原图是**抠的原始文件**（arxiv 源码包 / MinerU），不是糊截图。
- [ ] 公式用 LaTeX/KaTeX 渲染（唯一该"去截图"的）；自构流程用 Mermaid/TikZ。
- [ ] 重建的图（仅当有数据）注明"笔者据原文数据重绘"。
- [ ] diff 表、特性表、超参表该有的都有。

## C2. 实验讲透了吗（详见 experiments_section）
- [ ] 实验**单独成章**，Setup/主结果/消融/训练动态/现象图/结论六块齐全。
- [ ] **每张结果图都配解读**（"图里看到 X → 说明 Y"），没有光放图不讲。
- [ ] **消融到位**：读者能答"提升主要靠哪个改动"，且区分了提分/增稳/提效。
- [ ] 训练动态指标讲了"为什么重要 + 怎么读异常"（含过拟合等信号）。

## D. 结构与语气
- [ ] 章节顺序 = 简介→前置→主体(逐 diff)→实验→总结，且附 Paper Link。
- [ ] 中英术语并行；语气口语但精确；emoji 节制（每节 0–2）。
- [ ] 长节用 `---` 分「现象/根源/解法」，要点 bullet 化，无大段糊字。

## 一句话验收
> **"先讲透前置、每改动配 diff 表和为什么、每公式逐符号拆、每概念先直觉后数学、至少一个跑通的数字例子、论文关键原图一张不丢且每图配解读、实验单独成章且消融讲清提升靠哪、所有数字溯源原文。"** —— 全中才算一份 slothreport。
